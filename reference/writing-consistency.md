# 全篇写作一致性（Writing Consistency）

本文件定义跨 Module 的"风格基线"和"字数硬约束"，解决两个具体问题：
1. **差异化板块字数膨胀**：AI 在差异化角度落地时倾向"角度越独特 = 段落越长"，导致部分板块超字数上限 50%+
2. **风格漂移**：不同 Module 的 bullet 长度、段落节奏、数据呈现方式不一致，整篇读起来像几个人拼凑的

每个生成型 Module（1-13）都要遵循下方规则，且 Module 0 总策划结束时要输出一份"风格基线"摘要让后续 Module 锚定。

---

## 规则 1：全文 ≤2,000 词 + 每 Module 硬上限（零例外）

**核心**：boss 要求全文 **总字数 ≤2,000 词**（含表格全部单元格 + 正文 + bullet）。每 Module 都有反推自总预算的硬上限。

### 全文总预算

| 场景 | 总词数上限 |
|---|---|
| 必跑模块（AI 判 Module 4/7/8 后确定）| **~2,000 词** |
| 若 AI 判定加 Module 7 或 8 之一 | 需同步压缩其他模块 ≤2,100 |
| Module 7 + 8 全含 | 极不推荐，会突破 2,200 词 |

### 每 Module 硬上限（按 2,000 总预算反推）

| Module | 硬上限 | 计入范围 |
|---|---|---|
| 1 Hero | 55 (短) / 110 (长) | 正文 + bullet |
| 2 What Is | 180 | 正文 + bullet |
| 3 How Works | 180 | 正文 + bullet |
| 4 Types | **180** | 150 prose + 30 表格预算（4 行×5 列小表）|
| 5 Specs | **180** | 30 disclaimer + 150 双表预算（主表 ~70 + 矩阵 ~80）|
| 6 Applications | 250 | 正文 + bullet |
| 7 Project Case | **230** | 170 叙述 + 60 参数表 |
| 8 vs Comparison | **170** | 110 prose + 60 对比表 |
| 9 Selection Guide | 220 | 正文 + bullet |
| 10 Price Factors | 200 | 正文 + bullet |
| 11 Custom Solutions | 130 | 正文 + 询盘表 bullet |
| 12 Manufacturer | 170 | 2 段正文 |
| 13 FAQ | 240 总（每答 ≤30）| 8 Q&A 全部 |

**累计验算**：跳 7+8 → 55+180+180+180+180+250+220+200+130+170+240 = **1,985 词** ✓

### 表格字数计法（含表格 Module）

含表格的 Module（4 Types / 5 Specs / 7 Project Case / 8 vs）字数 = **正文 + bullet + 表头单元格 + 所有内容单元格 + 表格 disclaimer/intro**，全部按词数算，跟 Word 字数统计口径一致。空单元格不算。

**注意**：
- 如果 Module 5 只放 1 个主参数表 → 用 ~70 词表格预算 + ≤110 词 prose（总 180）
- 如果 Module 5 加差异化矩阵表（如 5物料×6维度）→ 主表 70 + 矩阵 80 + prose 30 = 180 词
- 矩阵表实测词数若超 80 → 减主表行数（从 7 行减到 5-6 行），不要扩 prose
- Module 7 表格 7-10 行 ≈ 60 词；超出 10 行的表格需相应砍叙述
- Module 8 表格 6-9 行 × 3 列 ≈ 60 词

### 写完即数
- 每个 Module 输出后立即在自检里报告字数（不写"略超"/"略多"等模糊词）
- 超出硬上限 → **必须重写**，不允许用"差异化角度复杂 / 数据密度需要"作为豁免理由
- 如果硬上限内确实塞不下所有数据点 → **移到表格**，不要往段落里塞

### 差异化角度的正确落地方式
- ❌ 错误：bullet 从 1 句 + 1 数据点 扩成 2 句 + 3 数据点
- ❌ 错误：段落从 1 段 + 3 句 扩成 2 段 + 6 句
- ✅ 正确：**保持 bullet/段落长度不变，把额外数据塞进表格** —— Module 4/5/8 都有现成表格位
- ✅ 正确：差异化角度只占 1 个 bullet + 1 个表格条目，不蔓延全段
- ✅ 正确：核心结论 1 句 + 1 个具体数据锚（XX-XX kcal/kg / XX% / XX 天），不展开"为什么"——展开放 FAQ

### 数据密度 ≠ 段落长度
**差异化深度由"每 100 词包含多少具体数据点"决定，不由"段落写多长"决定。**

参考密度（取整篇平均）：
- 优秀：每 100 词含 5-7 个具体数据点（带单位 / 区间 / 百分比 / 时间）
- 合格：每 100 词含 3-4 个
- 不合格：每 100 词 ≤2 个具体数据（说明段落在堆形容词或空话）

## 规则 2：跨 Module 风格一致性

**核心**：整篇产品页应该读起来像一个工程师从头写到尾，不是 13 个不同 AI 拼起来的。

### bullet 长度一致性
- 整篇 bullet 应在同一个长度区间内（如全部 12-25 词 / 全部 18-35 词）
- ❌ 错误：Module 4 的 bullet 平均 15 词，Module 6 的 bullet 平均 40 词
- ✅ 正确：Module 0 风格基线锁定后，所有 Module 的 bullet 长度对齐 ±30% 区间

### 段落节奏一致性
- 整篇段落平均 2-4 句 / 每句 18-30 词
- ❌ 错误：Module 2 写 3 句短段落，Module 9 写 5 句长段落
- ✅ 正确：所有 Module 的段落长度保持 ±25% 区间

### 数据呈现方式一致性
- 选定一种区间表示法后全篇统一：`50–10,000 t/d` 或 `50-10,000 t/d`（不混用 en dash 和 hyphen）
- 单位写法统一：`kcal/kg` 全篇统一（不混用 `kcal/kg` 和 `kcal per kg`）
- 数字格式统一：`1,400 °C` 全篇统一（不混用 `1400°C` 和 `1,400 °C`）

### em dash 密度一致性
- 沿用 [forbidden-words.md](forbidden-words.md) 已有规则：每段 ≤2 处段内 em dash + bullet 引导不限
- 跨 Module 应保持密度一致 —— 如果 Module 3 每段 1 处 em dash，Module 6 也应该 0-1 处，不应该突然 3 处

### 表格 vs 长 bullet 偏好
- 当一组数据 > 5 个对比维度 → 用**表格**，不用长 bullet
- 当一组数据 ≤ 3 个对比维度 → 用 bullet 即可
- ❌ 错误：用 8 行 bullet 描述 5 物料的 6 个参数（应该用表格）
- ✅ 正确：5 物料 × 6 参数 → 横向表格；3 物料 × 2 参数 → 3 个 bullet

## 规则 3：Module 0 必须输出"风格基线"

Module 0 总策划结尾要单独输出一段"风格基线"中文摘要，让后续 11 个生成型 Module 锚定：

```
═══════════════════════════
本篇风格基线（后续 Module 锚定）
═══════════════════════════

📏 字数硬上限（按 +15% 套用）：
- Module 1 Hero: 短版 ≤63 词 / 长版 ≤121 词
- Module 2-3 What Is + How Works: ≤207 词正文（不含 bullet）
- Module 4 Types: ≤173 词 + 表格
- Module 6 Applications: ≤288 词
- Module 9 Selection Guide: ≤253 词
- Module 10 Price: ≤230 词
- Module 11 Custom: ≤150 词 + 询盘表
- Module 12 Manufacturer: ≤195 词
- Module 13 FAQ: 8 × ≤30 词答案

✍️ 写作锚点：
- bullet 平均长度：[XX 词]（短/中/长，由本篇产品复杂度决定，跑前定）
- 段落平均：[X 句 / 段]，每句 [XX 词]
- 数据密度目标：每 100 词 ≥4 个具体数据点
- 数字表示：[选 en dash "–" 或 hyphen "-"]
- 单位表示：[首次出现时锁定，全篇沿用]

🎯 差异化角度处理原则：
- 角度落地优先放表格，不蔓延段落
- 每个 bullet 1 句 + 1 数据锚，不扩成 2 句
- 角度复杂度高 → 在 Module X 加独立表格，不在段落里展开
```

---

## 何时应用本规则

- **Module 0 跑完后**：必须输出"风格基线"
- **每个 Module 1-13 写完后**：自检字数 vs 硬上限 + bullet 长度 vs 基线 + 数据密度
- **超出硬上限 → 立即重写**，优先把内容移到表格而非压缩
- **自检报告输出时**：在"字数统计"表格里增加一列 `硬上限 (+15%)`，每行标 ✅/❌
